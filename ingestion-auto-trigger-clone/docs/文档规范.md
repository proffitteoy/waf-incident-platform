# 文档规范

## 目标

保障文档与代码一致，且可作为开发、联调、验收的直接依据。

## 基础规范

- 语言：中文为主，术语中英文可并列。
- 编码：UTF-8。
- 路径：使用仓库相对路径（如 `backend/src/api/router.ts`）。
- 时间：进度类文档必须写明绝对日期（如 `2026-02-25`）。

## 结构规范

每个文档至少包含：

- 文档目的
- 适用范围/边界
- 当前实现状态（已实现/未实现）
- 变更影响

## 同步规范

1. 路由或参数变更
- 必须同步：`api-spec.md`、`mcp-tools.md`。

2. 数据流或分层变更
- 必须同步：`architecture.md`、`canvas-dev/project.canvas`、`技术细节.md`。

3. 执行与回滚链路变更
- 必须明确说明：
  - 是否写入 `actions` 表
  - Redis 状态键写入策略
  - 回滚是否清理 Redis 键

4. 测试能力变更
- 必须同步：`testing.md`，并标注新增/删除用例。

## 禁止项

- 禁止保留旧路径（如 `vibe-codeing/*`）作为主引用。
- 禁止出现与当前代码冲突的流程描述。
- 禁止将"计划中的能力"写成"已实现"。
