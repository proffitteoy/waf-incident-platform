# 文档规范

## 目标

保证文档可直接作为 web-test 的开发、联调、验收依据。

## 基础规范

- 语言：中文为主。
- 编码：UTF-8。
- 路径：使用仓库相对路径。
- 时间：进度文档必须使用绝对日期（示例：`2026-03-05`）。

## 结构规范

每个文档至少包含：

- 文档目的
- 适用范围/边界
- 当前实现状态
- 变更影响

## 同步规范

1. 路由或参数变化
- 必须同步：`api-spec.md`、`技术细节.md`。

2. 数据流或分层变化
- 必须同步：`architecture.md`、`canvas-dev/project.canvas`。

3. 攻击场景变化
- 必须同步：`testing.md`、`waf-integration.md`。

## 禁止项

- 禁止保留过期路径或旧接口描述。
- 禁止将未实现的能力标记为已实现。
- 禁止文档与代码不一致。
