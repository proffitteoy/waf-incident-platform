# 强前置条件约束

### 通用开发约束

1. 不得为了演示攻击场景而执行真实破坏性系统命令。
2. 不得在演示代码中落地真实用户敏感信息。
3. 不得在核心业务逻辑中混入不可控副作用。
4. 不得吞异常，错误必须返回明确语义。
5. 不得让路由层承担复杂业务规则，逻辑应下沉到 services。
6. 不得在多个位置维护同一份事实数据。
7. 不得在未验证路由与文档一致性的情况下合并变更。
8. 不得将“计划能力”写成“已实现”。

### 胶水开发约束

1. 站点层仅负责页面与 API 编排，不重复实现 WAF 引擎能力。
2. 攻击检测以外部规则集为权威，本项目仅提供可复现输入场景。
3. 不得用 mock 结果替代真实 HTTP 请求路径验证。

### 文档与架构同步约束

1. 新增攻击场景后，必须同步 `api-spec.md`、`testing.md`、`waf-integration.md`。
2. 架构调整后，必须同步 `architecture.md` 与 `canvas-dev/project.canvas`。
3. 每个目录必须维护 README，至少包含：内容、边界、接口、入口。
