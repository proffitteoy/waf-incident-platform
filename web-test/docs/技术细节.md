# 技术细节（web-test）

## 文档目的

记录 web-test 的站点定位、实现状态与核心流程，作为迭代基线。

## 适用范围/边界

- 仅覆盖 `web-test` 子项目。
- 不覆盖主项目生产系统。

## 思路

把 web-test 设计为真实无业务内容站，并保留受控攻击入口用于 WAF 规则联调。

## 当前落地进度（2026-03-05）

### 已完成

- Express + EJS 内容站：`/`、`/about`、`/articles`、`/articles/:slug`。
- 攻击入口：
  - SQLi：`POST /api/auth/login`
  - XSS：`GET /search?q=...`
  - 命令注入探测：`POST /api/tools/ping-preview`
  - 路径穿越探测（附加）：`GET /legacy/download?name=../../etc/passwd`
- 自动化测试与回放文档已同步。

### 未完成（后续迭代）

- 未容器化打包。
- 未接入 E2E 浏览器测试。

## 核心流程

`Request -> Route -> Service -> View/Data -> Response`

启用 WAF 时：

`Request -> WAF/CRS -> web-test -> Response`

## 代码结构

- `src/server.js`：启动入口
- `src/app.js`：应用装配
- `src/routes`：页面路由与 API
- `src/services`：内容与攻击场景逻辑
- `src/views`：页面模板
- `data/files`：演示文件

## 变更影响

- 攻击入口变化：同步 `api-spec.md`、`testing.md`、`waf-integration.md`。
- 架构变化：同步 `architecture.md` 与 `canvas-dev/project.canvas`。
