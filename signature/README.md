# signature 板块说明

## 板块内容
- `rules/`：签名规则配置文件（参考用）。
- `reports/`：签名匹配报告与审计文件（参考用）。
- `backend/`：**取证模块参考实现**（功能完整，可独立部署，生产环境建议合并至主后端）。
- `services/`：取证 worker（Python 抓包进程，**生产环境统一使用根目录 `services/forensics-worker`**）。
- `docs/`：接口契约与技术文档（与主项目 `docs/` 保持一致）。

## 边界与定位
- **逻辑整合**：生产环境中，取证功能由主后端 (`backend/`) 统一提供，不独立启动 `signature/backend` 服务。
- **物理独立**：本目录保留完整代码结构，作为：
  1. **团队标准组件**：供团队成员查看取证模块的完整独立实现逻辑，代码已通过功能验证。
  2. **独立部署备选**：若未来需要拆分为独立微服务，可直接基于此目录部署。
  3. **Git 管理**：便于 Code Review 和版本追踪。

## 开发模式（重要）
- ✅ **当前任务**：仅在 `signature/` 目录内开发和修复报错，确保模块独立运行。
- ✅ **忽略外部**：无需关心根目录或其他目录的报错，那些由团队负责人在整合阶段处理。
- ✅ **交付标准**：确保 `signature/backend` 能独立 `npm run build` 且无 TS 报错。

✅ **异步闭环已打通**：
