## 取证链路（signature 增强版）

### 完整流程

1. **触发抓包**：`POST /api/incidents/:id/forensics/capture`
   - 验证事件单 ID
   - 计算抓包时间窗口
   - 创建 `forensics` 记录，状态 `queued`
   - 返回取证任务 ID

2. **Worker 执行**：`services/forensics-worker/src/capture.py`
   - 接收 `--forensics-id` 参数
   - 调用 `tshark` 抓包
   - 计算 `sha256` 和 `size_bytes`
   - **回写数据库**：更新 `forensics` 表状态为 `completed`

3. **下载文件**：`GET /api/forensics/:fid/download`
   - 查询 `forensics` 记录
   - 验证状态为 `completed`
   - 读取真实 pcap 文件
   - **实时计算 SHA256 签名**
   - 返回文件流 + `X-SHA256-Signature` 响应头

4. **元数据查询**：`GET /api/forensics/:fid/meta`
   - 返回取证记录元数据（不下载文件）

### 数据库表结构（forensics）

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| id | uuid | 主键 |
| incident_id | uuid | 关联事件单 |
| pcap_path | text | 文件存储路径 |
| sha256 | char(64) | 文件签名 |
| size_bytes | bigint | 文件大小 |
| status | text | queued/capturing/completed/failed |
| created_at | timestamptz | 创建时间 |
| capture_window_start | timestamptz | 抓包窗口开始 |
| capture_window_end | timestamptz | 抓包窗口结束 |
| completed_at | timestamptz | 完成时间 |
| error_message | text | 失败原因 |

### 启动说明

1. 确保 PostgreSQL 运行且 `forensics` 表已创建
2. 配置 `.env` 文件（参考 `.env.example`）
3. 启动后端：`cd signature/backend && npm run dev`
4. 触发抓包：`POST /api/incidents/{incident_id}/forensics/capture`
5. 执行 Worker：`python services/forensics-worker/src/capture.py --forensics-id {id}`
6. 下载文件：`GET /api/forensics/{fid}/download`
